初始組態包括下列步驟：

  1. 儲存系統組態

  2. 外掛程式安裝的認證設定

  3. 對於中央HANA外掛主機：

    1. 主機組態與SAP HANA外掛部署

    2. SAP HANA hdbsql用戶端軟體安裝與組態

  4. 原則組態

下列各節將說明初始組態步驟。

儲存系統組態

  1. 登入SnapCenter 到「伺服器GUI」。

  2. 選取儲存系統。

    在畫面中、您可以選擇儲存系統類型ONTAP 、即「SVM」或ONTAP 「叢集」。如果您在SVM層級上設定儲存系統、則需要為每個SVM設定管理LIF。此外、SnapCenter 您也可以在叢集層級使用「不二」管理存取。以下範例使用SVM管理。

  3. 按一下「New（新增）」以新增儲存系統、並提供所需的主機名稱和認證資料。

    SVM使用者不需要是vsadmin使用者、如螢幕擷取畫面所示。一般而言、使用者是在SVM上設定、並指派執行備份與還原作業所需的權限。如需必要權限的詳細資料、請參閱 "《安裝指南》SnapCenter" 在標題為ONTAP 「最低要求的權限」一節中。

  4. 按一下「更多選項」以設定儲存平台。

    儲存平台可以FAS 是不完整的、AFF 不完整的、ONTAP Select 不完整的、不完整的。Cloud Volumes ONTAP

    對於用作SnapVault SnapMirror或SnapMirror目標的系統、請選取次要圖示。

  5. 視需要新增其他儲存系統。在我們的範例中、我們新增了額外的異地備份儲存設備和災難恢復儲存設備。

認證設定

  1. 移至「設定」、選取「認證」、然後按一下「新增」。

  2. 提供使用者在Linux系統上用於外掛程式安裝的認證資料。

  3. 提供使用者在Windows系統上用於外掛程式安裝的認證資料。

    下圖顯示已設定的認證資料。

在中央外掛主機上安裝SAP HANA外掛程式

在實驗室設定中SnapCenter 、也會將此伺服器當作中央HANA外掛主機使用。執行效能不穩定伺服器的Windows主機SnapCenter 會新增為主機、SAP HANA外掛程式則安裝在Windows主機上。

SAP HANA外掛程式需要Java 64位元1.8版。在部署SAP HANA外掛程式之前、必須先在主機上安裝Java。

  1. 移至主機、然後按一下新增。

  2. 提供所需的主機資訊。按一下「提交」。

    下圖顯示部署HANA外掛程式後的所有已設定主機。

SAP HANA hdbsql用戶端軟體安裝與組態

SAP HANA hdbsql用戶端軟體必須安裝在安裝SAP HANA外掛程式的相同主機上。您可以從下載軟體 "SAP支援入口網站"。

在資源組態期間設定的HDBSQL OS使用者必須能夠執行hdbsql執行檔。hdbsql執行檔的路徑必須設定在「hana.properties`」檔案中。

  • Windows：

C:\More C:\Program Files\NetApp\SnapCenter\Snapcenter Plug-in Creator\etc\hana.properties
HANA_HDBSQL_CMD=C:\\Program Files\\sap\\hdbclient\\hdbsql.exe

  • Linux：

cat /opt/NetApp/snapcenter/scc/etc/hana.properties
HANA_HDBSQL_CMD=/usr/sap/hdbclient/hdbsql

原則組態

如一節所述 "「資料保護策略」" 原則通常會獨立設定資源、可供多個SAP HANA資料庫使用。

典型的最低組態包含下列原則：

  • 不需複寫的每小時備份原則：「本地管理單元」

  • 每日備份的原則SnapVault （含資料複寫）：「LocalSnapAndSnapVault」（本地SnapAndSnapVault）

  • 使用檔案型備份進行每週區塊完整性檢查的原則：「BlockIntegrityCheck」

以下各節將說明這三項原則的組態。

每小時Snapshot備份原則

  1. 移至「設定」>「原則」、然後按一下「新增」。

  2. 輸入原則名稱和說明。按一下「下一步」

  3. 選取備份類型做為「Snapshot Based」（快照型）、然後選取「Hourly」（每小時）

  4. 設定隨需備份的保留設定。

  5. 設定排程備份的保留設定。

  6. 設定複寫選項。在這種情況下、未SnapVault 選取任何更新功能。

  7. 在「摘要」頁面上、按一下「完成」。

每日Snapshot備份與SnapVault 還原複寫的原則

  1. 移至「設定」>「原則」、然後按一下「新增」。

  2. 輸入原則名稱和說明。按一下「下一步」

  3. 將備份類型設為「Snapshot Based（根據Snapshot）」、排程頻率設為「每日」。

  4. 設定隨需備份的保留設定。

  5. 設定排程備份的保留設定。

  6. 建立本機Snapshot複本後、請選取「更新SnapVault 」。

    次要原則標籤必須與儲存層資料保護組態中的SnapMirror標籤相同。請參閱一節 "「將資料保護組態設定為異地備份儲存設備。」"

  7. 在「摘要」頁面上、按一下「完成」。

每週區塊完整性檢查原則

  1. 移至「設定」>「原則」、然後按一下「新增」。

  2. 輸入原則名稱和說明。按一下「下一步」

  3. 將備份類型設為「檔案型」、並將排程頻率設為「每週」。

  4. 設定隨需備份的保留設定。

  5. 設定排程備份的保留設定。

  6. 在「摘要」頁面上、按一下「完成」。
